# Requirements: Investor Showcase App

**Defined:** 2026-03-03
**Core Value:** High-fidelity, frictionless demonstration of resident registration and instant ID generation with a premium Neon Mint UI.

## v1 Requirements

Requirements for the initial investor showcase. Focused on the "Registration → Dashboard → ID" flow.

### Visual Identity & Layout (UI)

- [ ] **UI-01**: Implement Neon Mint (#00FFA3) and Zinc/Deep Slate palette from reference dashboard.
- [ ] **UI-02**: Implement 256px Sidebar and Sticky Header admin dashboard layout.
- [ ] **UI-03**: Dashboard displays premium metric cards (Active Residents, IDs Issued, etc.).
- [ ] **UI-04**: Implement Framer Motion transitions for smooth UI/UX flow.

### Registration & Onboarding (REG)

- [ ] **REG-01**: Implement modern multi-step registration form.
- [ ] **REG-02**: Relax form validation to make non-critical parameters optional for demo.
- [ ] **REG-03**: Add "Demo Auto-fill" button for rapid onboarding during presentations.

### Dashboard & Navigation (DASH)

- [ ] **DASH-01**: Create clean dashboard landing view directing users to ID generation.
- [ ] **DASH-02**: Implement responsive sidebar navigation for desktop and mobile.

### ID Generation (IDGEN)

- [ ] **IDGEN-01**: Refine Canvas-based ID generation logic for high visual quality.
- [ ] **IDGEN-02**: Implement instant client-side ID preview (Front/Back) after registration.
- [ ] **IDGEN-03**: Enable high-resolution ID card export (Image or PDF).
- [ ] **IDGEN-04**: Generate unique QR codes for each ID card.

## v2 Requirements

Deferred features for future development.

### Advanced Features

- **VERI-01**: Real-time server-side ID verification system.
- **WALL-01**: Apple/Google Wallet integration for IDs.
